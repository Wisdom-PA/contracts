API definitions shared by the cube and the mobile app: REST or WebSocket shapes, request/response schemas. Optionally app ↔ backend.

- **Plan and tickets:** See the [listener](https://github.com/Wisdom-PA/listener) repo.

Add `openapi.yaml` (or similar) and document consumption in this README once the cube ↔ app API is defined (Phase 1.4).
